Since 1.9.0, MarkInside provides a local **MCP (Model Context Protocol) server**. You can either connect to this MCP server directly, or install a plugin that wraps the connection flow.

## Connet to the MCP Server Directly

By default, the server listens on port **9448**.

You can check or change the port in **MarkInside → Settings → MCP Server**.

Replace `<PORT>` with your actual port number (use `9448` if unchanged).

<Tabs syncKey="agent">
<TabItem label="Codex">

Add this to your Codex configuration file:

- User-level: `~/.codex/config.toml`
- Project-level: `.codex/config.toml`

```toml
[mcp_servers.markinside]
url = "http://127.0.0.1:<PORT>/mcp"
```

</TabItem>
<TabItem label="Claude Code">

Register the MCP server:

```bash
claude mcp add --transport http markinside http://127.0.0.1:<PORT>/mcp
```

For project scope, run:

```bash
claude mcp add --transport http --scope project markinside http://127.0.0.1:<PORT>/mcp
```

</TabItem>
</Tabs>

## Connect through Agent Plugin

You can also use MarkInside through an agent plugin: it still talks to the same MCP server, but packages MCP config and workflow skills together.

<Tabs syncKey="agent">
<TabItem label="Codex">

Plugin structure:

<FileTree>
- markinside/
  - .codex-plugin/
    - plugin.json
  - .mcp.json
  - skills/
    - markinside/
      - SKILL.md
</FileTree>

`markinside/.codex-plugin/plugin.json`:

```json
{
  "name": "markinside",
  "version": "0.1.0",
  "description": "Skills and MCP config for MarkInside workflows.",
  "skills": "./skills/",
  "mcpServers": "./.mcp.json"
}
```

`markinside/.mcp.json`:

```json
{
  "mcpServers": {
    "markinside": {
      "url": "http://127.0.0.1:<PORT>/mcp"
    }
  }
}
```

`markinside/skills/markinside/SKILL.md`:

```md
---
name: markinside
description: Use MarkInside MCP tools to create and edit code images.
---

Always use the MarkInside MCP tools first when users ask for image generation or edits in MarkInside.
```

Register the plugin in a marketplace file:
- Project scope: `$REPO_ROOT/.agents/plugins/marketplace.json`
- Personal scope: `~/.agents/plugins/marketplace.json`

```json
{
  "name": "markinside-local-plugins",
  "interface": {
    "displayName": "MarkInside Local Plugins"
  },
  "plugins": [
    {
      "name": "markinside",
      "source": {
        "source": "local",
        "path": "./plugins/markinside"
      },
      "policy": {
        "installation": "AVAILABLE"
      },
      "category": "Productivity"
    }
  ]
}
```

If you use personal scope, place plugin files under `~/.codex/plugins/markinside` and set `path` to `./.codex/plugins/markinside`.

Restart Codex, open `/plugins`, choose your local marketplace, and install `markinside`.

</TabItem>

During development, load it directly:

```bash
claude --plugin-dir ./markinside
```

For persistent installation, create a local marketplace in your repo (or personal plugin repo), then add and install it:

```text
/plugin marketplace add ./path/to/your-marketplace-repo
/plugin install markinside@<marketplace-name>
```

Use `claude plugin install markinside@<marketplace-name> --scope project` for project scope, or omit `--scope` for user scope.
